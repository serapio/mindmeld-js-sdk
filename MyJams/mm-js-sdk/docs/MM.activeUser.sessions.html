Namespace: sessions

MM.activeUser. sessions

MM.activeUser.sessions represents the user's sessions collection in the MindMeld API.

Methods

delete(sessionid, onSuccess, onFail)

Delete a session from the application and the user's session list

Name Type Description
sessionid string

id of the session to delete
onSuccess APISuccessCallback optional

callback for when deleting object was successful
onFail APIErrorCallback optional

callback for when deleting object failed
Example
         function deleteSession () {
            MM.activeUser.sessions.delete('72798', onSessionDeleted);
         }
         function onSessionDeleted (response) {
            // session deleted
         }

get(params, onSuccess, onFail)

Get the list of sessions that can be accessed by the specified user. A request made with a user token is permitted to get the session list for only the user associated with the token. A request made with an admin token can get the session list for any user of your application.

Name Type Description
params QueryParameters optional

A QueryParameters object allowing you to filter the sessions returned. See documentation here for more details
onSuccess APISuccessCallback optional

callback for when getting the session list was successful
onFail APIErrorCallback optional

callback for when getting the session list failed
Example
         function getSessions () {
            MM.activeUser.sessions.get(null, onGetSessions);
         }
         function onGetSessions (response) {
            var sessions = response.data;
         }

json(){Array.<Object>}

Helper function returns the JSON data for the sessions collection

Example
         function getSessions () {
            MM.activeUser.sessions.get(null, onGetSessions);
         }
         function onGetSessions (response) {
            var sessions = MM.activeUser.sessions.json();
            // MM.activeUser.sessions.json() returns a JSON object
            // containing data received from MM.activeUser.sessions.get()
         }

onUpdate(updateHandler, onSuccess, onError, getQueryParams)

Sets the activeUser.session's onUpdate handler. If no handler is passed in, onUpdate unsubscribes from push events

Name Type Description
updateHandler APISuccessCallback optional

callback for when the active user's session list updates
onSuccess function optional

callback for when subscription to onUpdate event succeeds
onError function optional

callback for when subscription to onUpdate event fails
getQueryParams QueryParamGetter optional

custom function used to determine QueryParameters used to in get() request when collection updates
Examples

Setting the onUpdate handler, creating a new session, and obtaining the latest session list 

         function sessionsOnUpdateExample () {
            // set the onUpdate handler for the sessions list
            MM.activeUser.sessions.onUpdate(
                onSessionsUpdate,
                onSubscribedToSessionsUpdates,
                onSubscribeToSessionUpdatesError,
                getSessionListParams
            );
         }
         function onSubscribedToSessionsUpdates () {
            // successfully subscribed to updates to the user's sessions list
            // now, create a new session
            createNewSession();
         }
         function onSubscribeToSessionUpdatesError () {
            console.log('error subscribing to session list updates');
         }
         function getSessionListParams () {
            // When the session list updates, only fetch 5 objects
            return {
                limit: 5
            };
         }
         function onSessionsUpdate () {
            // there was an update to the sessions list
            var sessions = MM.activeUser.sessions.json();
            // sessions contains the latest list of sessions
         }
         function createNewSession () {
            var newSessionData = {
                name: 'new session name',
                privacymode: 'inviteonly'
            };
            MM.activeUser.sessions.post(newSessionData);
         }

Deregistering the onUpdate handler 

         function deregisterSessionListOnUpdate () {
            MM.activeUser.sessions.onUpdate(null);
         }

post(sessionInfo, onSuccess, onFail)

Creates a new session for currently active user. This will create a new session, and the specified user will be set as the session organizer, as indicated by the 'organizer' attribute of the session object. A request made with a user token is permitted to post to the session list for only the user associated with the token. A request made with an admin token is permitted to create a new session on behalf of any user of the application.

Name Type Description
sessionInfo Object

Object containing new session data. Please refer to documentation for creating sessions here for more info

Name Type Description
name string

name of the new session
privacymode string

the privacy mode for the session. The supported privacy modes are 'friendsonly', 'inviteonly', and 'public'. Sessions that are 'inviteonly' can be accessed only by the session organizer and any user on the inviteduser list for the session. Sessions that are 'friendsonly' can be accessed by users who are in the friends collection of the session organizer. Sessions that are 'public' can be accessed by all users of your application.
onSuccess APISuccessCallback optional

callback for when creating new session was successful
onFail APIErrorCallback optional

callback for when creating new session failed
Example
         function createNewSession () {
            var newSessionData = {
                name: 'new session name',
                privacymode: 'inviteonly'
            };
            MM.activeUser.sessions.post(newSessionData, onCreateNewSession);
         }
         function onCreateNewSession (result) {
            console.log(result);
         }
